# 在线阅卷诊断系统 - 需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
面向学校教师的在线阅卷诊断系统，支持教师对学生试卷进行高效批改、评分和反馈。

### 1.2 目标用户
- 中小学及高中教师
- 教研组长
- 学校管理人员

### 1.3 核心价值
- 提高批改效率：支持按题批改，同题连批
- 灵活批改方式：支持按学生、按题目两种批改模式
- 个性化反馈：支持卷面批注和学生诊断报告
- 报告管理：统一管理和发布学生诊断报告
- 效果预览：模拟学生端查看批改结果

## 2. 系统结构

### 2.1 全局导航
系统采用左侧固定导航栏，包含三个主要模块：

| 导航项 | 图标 | 模块ID | 说明 |
|--------|------|--------|------|
| 阅卷诊断 | 📝 | moduleGrading | 核心批改功能，包含诊断列表和批改页面 |
| 诊断管理 | 📊 | moduleDiagnosis | 管理诊断配置、报名名单、咨询名单、诊断报告 |
| 学生平板效果 | 📱 | moduleStudent | 预览学生端查看成绩的效果 |

### 2.2 数据流转关系

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           诊断管理模块                                    │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                  │
│  │  创建诊断   │───▶│  发布诊断   │───▶│ 学生报名    │                  │
│  │  配置       │    │  开放入口   │    │ /数据上传   │                  │
│  └─────────────┘    └─────────────┘    └──────┬──────┘                  │
│                                               │                          │
│         ┌─────────────────────────────────────┘                          │
│         ▼                                                                │
│  ┌─────────────┐                                                         │
│  │ 报名名单    │ ◀── 记录报名学生和答题状态                              │
│  └─────────────┘                                                         │
└─────────────────────────────────────────────────────────────────────────┘
         │
         │ 已报名+已提交/已上传答题卡的学生
         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           阅卷诊断模块                                    │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                  │
│  │  诊断列表   │───▶│  批改页面   │───▶│ 诊断报告    │                  │
│  │  (按类型)   │    │ (批注+评分) │    │ (发布)      │                  │
│  └─────────────┘    └─────────────┘    └──────┬──────┘                  │
└─────────────────────────────────────────────────────────────────────────┘
         │
         │ 报告发布后更新状态
         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           诊断管理模块                                    │
│  ┌─────────────┐    ┌─────────────┐                                     │
│  │ 诊断报告    │    │ 咨询名单    │ ◀── 学生查看报告后提交咨询          │
│  │ 列表        │    │             │                                     │
│  └─────────────┘    └─────────────┘                                     │
└─────────────────────────────────────────────────────────────────────────┘
```

**数据流转说明：**

1. **诊断配置 → 阅卷列表**
   - 诊断管理中创建并发布的诊断，会出现在阅卷诊断列表中
   - 只有状态为"进行中"或"已结束"的诊断才会显示在阅卷列表

2. **报名数据 → 阅卷范围**
   - 在线模考：已报名且已提交答题的学生进入阅卷范围
   - 数据直传：已报名且已上传答题卡的学生进入阅卷范围

3. **阅卷完成 → 状态同步**
   - 阅卷模块发布报告后，报名名单中对应学生状态更新为"报告已发布"
   - 诊断报告列表同步新增该学生的报告记录

## 4. 业务流程

### 4.1 诊断配置流程

```
创建诊断配置 → 填写基本信息 → 配置时间 → 配置题目 → 选择学生范围
→ 保存草稿/发布诊断
```

### 4.2 学生参与流程

**在线模考：**
```
学生报名 → 等待考试开始 → 在线答题 → 提交答卷 → 等待阅卷 → 查看报告
```

**数据直传：**
```
学生报名 → 线下考试 → 管理员上传答题卡和小分表 → 教师阅卷批注 → 查看报告
```

### 4.3 教师阅卷流程

**按题批改：**
```
选择诊断 → 点击"按题诊断" → 选择题目 → 批改学生A该题 → 提交
→ 自动切换到学生B该题 → ... → 该题所有学生批完 → 选择下一题
```

**按学生批改：**
```
选择诊断 → 点击"按学生诊断" → 选择学生 → 批改第1题 → 提交并去下一题
→ 批改第2题 → ... → 完成最后一题 → 弹出诊断报告 → 编辑评价和寄语
→ 发布报告 → 自动切换到下一个学生
```

### 4.4 特殊场景处理

| 场景 | 处理方式 |
|------|----------|
| 点击"待发布"状态学生 | 直接弹出可编辑报告 |
| 点击"已完成"状态学生 | 弹出只读报告 |
| 发布报告后下一个学生也是"待发布" | 自动弹出该学生报告 |
| 数据直传模式批改 | 隐藏评分输入，显示自动获取的分数 |

## 6. 异常处理与边界情况

### 6.1 时间校验

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 报名结束时间 | 必须晚于报名开始时间 | "报名结束时间必须晚于开始时间" |
| 考试开始时间 | 不能早于报名开始时间 | "考试开始时间不能早于报名开始时间" |
| 考试结束时间 | 不能晚于报名结束时间 | "考试结束时间不能晚于报名结束时间" |
| 考试结束时间 | 必须晚于考试开始时间 | "考试结束时间必须晚于开始时间" |

### 6.2 必填字段校验

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 诊断名称 | 非空 | "请输入诊断名称" |
| 诊断科目 | 已选择 | "请选择诊断科目" |
| 报名开始时间 | 非空 | "请选择报名开始时间" |
| 报名结束时间 | 非空 | "请选择报名结束时间" |
| 考试开始时间 | 非空（在线模考） | "请选择考试开始时间" |
| 考试结束时间 | 非空（在线模考） | "请选择考试结束时间" |
| 题目列表 | 至少1题 | "请至少添加一道题目" |
| 学生范围 | 至少1人 | "请至少选择一名学生" |

### 6.3 操作确认

| 操作 | 确认提示 |
|------|----------|
| 清除所有批注 | "确定要清除所有批注吗？" |
| 发布诊断 | "确定发布诊断吗？发布后学生即可开始报名" |
| 删除诊断（待开始） | "确定删除该诊断配置吗？" |
| 批量关闭云控 | "确定关闭所有机构的诊断入口吗？" |

### 6.4 阅卷范围边界

| 场景 | 是否进入阅卷范围 | 说明 |
|------|------------------|------|
| 在线模考 - 已报名已提交 | ✓ 进入 | 正常流程 |
| 在线模考 - 已报名未提交 | ✗ 不进入 | 学生未完成答题 |
| 在线模考 - 未报名 | ✗ 不进入 | 学生未参与 |
| 数据直传 - 已报名已上传答题卡 | ✓ 进入 | 正常流程 |
| 数据直传 - 已报名未上传答题卡 | ✗ 不进入 | 缺少答题卡数据 |
| 数据直传 - 未报名已上传答题卡 | ✗ 不进入 | 学生未报名不参与 |

## 7. 非功能需求

### 7.1 性能要求
- 页面加载时间 < 2秒
- 批注操作响应时间 < 100ms
- 支持单次加载 50+ 份试卷
- 答题卡图片加载支持懒加载

### 7.2 兼容性
- 支持 Chrome、Firefox、Safari、Edge 最新版本
- 支持 1280px 及以上屏幕宽度
- 移动端暂不支持（建议横屏平板）

### 7.3 可用性
- 操作反馈及时（Toast 提示）
- 支持键盘快捷操作（ESC 关闭弹窗）
- 批注支持拖拽调整位置
- 涂鸦支持自由绘制
- 表单校验即时反馈
