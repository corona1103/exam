# 在线阅卷诊断系统 - 需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
面向学校教师的在线阅卷诊断系统，支持教师对学生试卷进行高效批改、评分和反馈。

### 1.2 目标用户
- 中小学及高中教师
- 教研组长
- 学校管理人员

### 1.3 核心价值
- 提高批改效率：支持按题批改，同题连批
- 灵活批改方式：支持按学生、按题目两种批改模式
- 个性化反馈：支持卷面批注和学生诊断报告
- 报告管理：统一管理和发布学生诊断报告
- 效果预览：模拟学生端查看批改结果
- 用户管理：管理教师和专家账号，支持角色权限控制
- 讲座排课：日历式讲座管理，支持外部专家入驻

### 2.3 三端状态对应关系

| 场景 | 诊断管理-报名名单 | 阅卷诊断-学生列表 | 学生平板-诊断卡片 |
|------|------------------|------------------|------------------|
| 学生报名后 | 未提交(notsubmit) | - | 已报名等待/可开考 |
| 学生提交答卷 | 阅卷中(grading) | 待诊断(none) | 待报告 |
| 教师开始批改 | 阅卷中(grading) | 诊断中(grading) | 待报告 |
| 教师批改完成 | 阅卷中(grading) | 待发布(pending) | 待报告 |
| 教师发布报告 | 报告已发布(published) | 已完成(published) | 报告已出 |

### 2.4 数据流转说明

**1. 诊断配置 → 多端生效**
- 诊断管理中创建并发布的诊断：
  - 对学生端：出现在诊断列表中，学生可报名参与
  - 对阅卷端：状态为"进行中"或"已结束"时显示在阅卷列表

**2. 学生行为 → 数据同步**
- 学生报名：
  - 报名名单新增记录，答题状态为"未提交"
  - 学生端卡片变为"已报名等待"
- 学生提交答卷（在线模考）或管理员上传答题卡（数据直传）：
  - 报名名单答题状态更新为"阅卷中"
  - 该学生进入阅卷范围，出现在批改页面学生列表中
  - 学生端卡片变为"待报告"

**3. 阅卷完成 → 全局状态同步**
- 教师发布报告后：
  - 报名名单：答题状态更新为"报告已发布"
  - 诊断报告列表：新增该学生的报告记录
  - 学生平板：诊断卡片变为"报告已出"，可查看完整报告

**4. 学生反馈 → 咨询流程**
- 学生查看报告后提交课程咨询：
  - 咨询名单新增记录
  - 教师可通过诊断管理模块查看和跟进

#### 3.1.2 阅卷规则说明

**客观题与主观题阅卷方式：**

| 诊断类型 | 题目类型 | 阅卷方式 | 说明 |
|----------|----------|----------|------|
| 在线模考 | 客观题（选择、判断、填空） | **系统自动阅卷** | 系统根据标准答案自动判分，无需人工干预 |
| 在线模考 | 主观题（解答题） | **人工阅卷批注** | 需要教师人工批改、添加批注、评分 |
| 数据直传 | 客观题 | **小分表导入** | 分数直接从上传的小分表中获取，无需人工批改 |
| 数据直传 | 主观题 | **人工阅卷批注** | 需要教师在答题卡图片上添加批注（分数从小分表获取） |

**阅卷范围：**
- 教师进入阅卷页面时，**只显示需要人工批改的主观题**
- 客观题自动完成阅卷，不出现在批改列表中
- 批改进度统计仅计算主观题的完成情况

#### 3.3.6 学生端业务流程

**在线模考完整流程**

```
诊断列表（未报名）
    │
    ▼ 点击"立即报名"
诊断列表（已报名-等待开考）
    │
    ▼ 考试时间到
诊断列表（可以开考）
    │
    ▼ 点击"开始考试" → 弹出确认弹窗 → 确定
考试界面
    │
    ├─▶ 点击"返回" → 离开确认弹窗 → 暂时离开 → 诊断列表（考试中）
    │
    ├─▶ 倒计时结束 → 自动提交弹窗 → 查看结果 → 诊断报告页
    │
    └─▶ 点击"交卷" → 提交确认弹窗 → 确认提交
            │
            ▼
    诊断列表（等待报告）
            │
            ▼ 老师发布报告
    诊断列表（报告已出）
            │
            ▼ 点击"查看报告"
    诊断报告页
            │
            └─▶ 填写课程咨询 → 提交 → 进入咨询名单
```

**边界情况处理**

| 场景 | 处理方式 |
|------|----------|
| 考试时间到未报名 | 卡片变为"已过期"状态，不可操作 |
| 考试中途离开再返回 | 显示"继续考试"按钮，保留答题进度 |
| 倒计时最后5分钟 | 倒计时数字变红色，添加警告样式 |
| 时间到自动提交 | 弹出自动提交弹窗，无需用户确认 |
| 报名已满 | 卡片显示"报名已满"，按钮禁用 |
| 手机号格式错误 | 表单校验提示"请输入正确的手机号" |

### 4.2 诊断配置流程

```
┌──────────────────────────────────────────────────────────────────────────────────────────┐
│                                诊断配置创建流程                                           │
└──────────────────────────────────────────────────────────────────────────────────────────┘

[创建诊断]
    │
    ├─── 基本信息 ──────────┬─── 诊断名称（必填）
    │                       ├─── 诊断科目（必填）
    │                       ├─── 诊断类型（在线模考/数据直传）
    │                       └─── 诊断说明（选填，≤100字）
    │
    ├─── 时间配置 ──────────┬─── 报名开始时间（必填）
    │                       ├─── 报名结束时间（必填，> 开始时间）
    │                       │
    │                       └─── 【仅在线模考】
    │                            ├─── 考试开始时间（必填，≥ 报名开始）
    │                            ├─── 考试结束时间（必填，≤ 报名结束）
    │                            └─── 考试限时（必填，单位：分钟）
    │
    ├─── 题目配置 ──────────┬─── 【在线模考】题目ID列表 + 分值设置
    │                       └─── 【数据直传】题目ID列表（分值从小分表导入）
    │
    ├─── 数据上传 ──────────┬─── 【仅数据直传】
    │   （数据直传）        ├─── 小分表上传 → 分数自动关联
    │                       └─── 答题卡上传 → 图片批注
    │
    ├─── 学生范围 ──────────── 三级级联选择（机构 → 班级 → 学生）
    │
    └─── 课程咨询配置 ──────┬─── 咨询说明文案
                            └─── 是否收集手机号
    │
    ▼
[保存草稿] ─── 暂存配置，可继续编辑
    │
    或
    │
    ▼
[发布诊断] ─── 诊断生效
    │
    ├─── 诊断状态变为"待开始"（当前时间 < 报名开始时间）
    ├─── 到达报名开始时间后，状态自动变为"进行中"
    └─── 通过云控开关控制各机构入口开放状态
```

| 场景 | 处理方式 |

## 6. 异常处理与边界情况

### 6.1 时间校验

**设计目标：**
- 支持提前预告和报名：报名开始时间可以早于考试开始时间
- 支持考试期间直接参与：考试期间未报名的学生也能直接报名并考试

**时间关系图：**
```
报名开放区间：|-------- 报名开始 ========== 报名结束 --------|
考试开放区间：           |--- 考试开始 ==== 考试结束 ---|

典型场景1（提前报名）：
  报名：4/10 ─────────────────────────── 4/20
  考试：        4/15 ────────── 4/18
  说明：学生可在4/10-4/14提前报名，4/15-4/18参加考试

典型场景2（随到随考）：
  报名：4/15 ─────────────────────────── 4/20
  考试：4/15 ─────────────────────────── 4/20
  说明：报名和考试同时开放，学生可随时报名并立即考试
```

**校验规则：**

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 报名结束时间 | 必须晚于报名开始时间 | "报名结束时间必须晚于开始时间" |
| 考试结束时间 | 必须晚于考试开始时间 | "考试结束时间必须晚于开始时间" |
| 报名结束时间 | 必须 ≥ 考试结束时间 | "报名结束时间不能早于考试结束时间，需保证考试期间学生可报名参与" |

**不再限制：**
- ~~考试开始时间不能早于报名开始时间~~ — 允许灵活配置
- ~~考试结束时间不能晚于报名结束时间~~ — 改为相反规则

### 6.2 必填字段校验

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 诊断名称 | 非空 | "请输入诊断名称" |
| 诊断科目 | 已选择 | "请选择诊断科目" |
| 报名开始时间 | 非空 | "请选择报名开始时间" |
| 报名结束时间 | 非空 | "请选择报名结束时间" |
| 考试开始时间 | 非空（在线模考） | "请选择考试开始时间" |
| 考试结束时间 | 非空（在线模考） | "请选择考试结束时间" |
| 题目列表 | 至少1题 | "请至少添加一道题目" |
| 学生范围 | 至少1人 | "请至少选择一名学生" |

### 6.3 操作确认

| 操作 | 确认提示 |
|------|----------|
| 清除所有批注 | "确定要清除所有批注吗？" |
| 发布诊断 | "确定发布诊断吗？发布后学生即可开始报名" |
| 删除诊断（待开始） | "确定删除该诊断配置吗？" |
| 批量关闭云控 | "确定关闭所有机构的诊断入口吗？" |

### 6.4 阅卷范围边界

| 场景 | 是否进入阅卷范围 | 说明 |
|------|------------------|------|
| 在线模考 - 已报名已提交 | ✓ 进入 | 正常流程 |
| 在线模考 - 已报名未提交 | ✗ 不进入 | 学生未完成答题 |
| 在线模考 - 未报名 | ✗ 不进入 | 学生未参与 |
| 数据直传 - 已报名已上传答题卡 | ✓ 进入 | 正常流程 |
| 数据直传 - 已报名未上传答题卡 | ✗ 不进入 | 缺少答题卡数据 |
| 数据直传 - 未报名已上传答题卡 | ✗ 不进入 | 学生未报名不参与 |
